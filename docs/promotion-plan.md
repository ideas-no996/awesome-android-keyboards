# Promotion Plan

Use restrained, community-specific posts. The goal is to invite corrections and contributors, not to claim authority over the Android keyboard ecosystem.

## Suitable Communities

- GitHub: Android, keyboard, IME, Rime, and awesome-list maintainers.
- F-Droid Forum: useful for privacy-focused and open-source keyboard discovery.
- Reddit `r/androiddev`: development angle, Android IME architecture, InputMethodService resources.
- Reddit `r/fossdroid`: open-source and F-Droid keyboard angle.
- Rime community spaces: Trime, Rime schemas, and Android Rime front-ends.
- Fcitx community spaces: Fcitx5 for Android and multilingual IME discussion.
- V2EX: Android, 输入法, Rime, 开源项目 nodes.
- 少数派 Matrix or personal blogs: curated Android input method comparison angle.
- X/Twitter or Mastodon: concise release note with repository link.
- Hacker News "Show HN": only after data quality improves beyond the initial seed.

## Short English Post

```text
I started Awesome Android Keyboards, a curated Android keyboard / Android IME ecosystem list.

It tracks commercial keyboards, open-source keyboards, privacy keyboard options, Rime and Trime projects, F-Droid packages, theme/customization references, and Android InputMethodService development resources.

The project is data-first: entries live in YAML, README tables are generated, and each item needs sources, maintenance status, privacy/network notes, and a short original review.

Corrections and additions are welcome:
https://github.com/ideas-no996/awesome-android-keyboards
```

## Release Timing

1. Publish quietly after the first public push.
2. Fix obvious metadata issues from early feedback.
3. Add more F-Droid and Rime ecosystem entries.
4. Publish to broader communities only after README, data schema, and contribution rules feel stable.

## Tone Guidelines

- Say "curated list" rather than "definitive list".
- Ask for corrections, especially around maintenance status and privacy claims.
- Do not compare projects with inflammatory language.
- Do not imply endorsement of proprietary keyboards.
- Keep posts short and link to contribution rules.
